### 1. Вывод данных: функция print() и её параметры

Функция `print()` выводит текст на экран. По умолчанию она разделяет аргументы пробелом и автоматически переносит строку в конце.

- `sep` (separator) — задаёт символ-разделитель между элементами (вместо пробела).
    
- `end` — задаёт символ, который пишется в самом конце строки (вместо переноса на новую строку `\n`).
    

Python

```
print("Anti", "Gremlin", sep="-", end=" ")
print("System")
# Выведет в одну строку: Anti-Gremlin System
```

### 2. Ввод данных и переменные: функция input()

- `input()` — останавливает выполнение программы и ждёт, пока пользователь введёт текст.
    
- **Критически важно:** `input()` всегда возвращает строку (`str`). Если нам нужно работать с числами, её обязательно нужно оборачивать в функцию преобразования типов: `int()` для целых чисел или `float()` для дробных.
    

Python

```
age = int(input("Введите ваш возраст: "))  # Строка преобразуется в целое число
```

### 3. Стандарт PEP 8 (Важно для командной разработки)

`PEP 8` — это официальный стиль оформления кода на Python. В ИТ-индустрии за "грязный" код не пропустят ни одно код-ревью.

- **Имена переменных:** должны быть говорящими, строчными буквами, слова разделяются подчеркиванием (`snake_case`), например: `user_payment_status`.
    
- **Пробелы:** вокруг операторов (`=`, `+`, `-`) всегда ставится по одному пробелу.
    
- **Отступы:** для вложенного кода всегда используются 4 пробела.

### 4. Инженерные заметки по безопасности и бэкенду

- **Валидация входящих типов (Защита от падения бэкенда):** Когда пользователь CRM-системы вводит стоимость абонемента или количество дней, бэкенд ожидает число. Если написать `days = int(input())`, а пользователь случайно введёт текст "десять", программа упадет с ошибкой `ValueError`. Любые данные от пользователя — это потенциальный яд. Перед приведением к `int()` их всегда нужно проверять, иначе злоумышленник сможет легко уронить сервер.
    
- **Утечка данных через логирование (ИБ-нюанс):**
    
    Функция `print()` часто используется для отладки. Но в реальном бэкенде всё, что выводится, уходит в системные логи (журналы сервера). **Никогда** нельзя выводить в `print()` sensitive-данные (пароли пользователей, хэши, данные банковских карт), иначе они осядут в текстовых файлах логов на сервере, к которым может получить доступ злоумышленник при взломе.
