# Тема 5.1: VLAN (Virtual Local Area Network)

Изучение технологии виртуальных локальных сетей, позволяющей разделять единый широковещательный домен на изолированные сегменты, механизмов тегирования (IEEE 802.1Q), режимов портов (Access/Trunk), маршрутизации между VLAN, Native VLAN, а также практических аспектов безопасности и масштабирования.

---

## 1. VLAN — Базовые свойства

- **Уровень OSI:** Канальный уровень (L2) для сегментации; маршрутизация между VLAN — сетевой уровень (L3).
- **Сущность данных (PDU):** Кадр (Frame).
- **Принцип работы:** VLAN (Virtual Local Area Network) позволяет разделить единую сеть (широковещательный домен) на изолированные друг от друга сегменты сети. Это повышает безопасность и уменьшает нагрузку от широковещательного трафика.

**Пример:** В бизнес-центре сдаются в аренду несколько офисов. В одном офисе работает компания с разработчиками, а в соседнем — туристическая фирма. Чтобы изолировать сети разных компаний, администратор назначает портам коммутатора разные **VLAN ID** (идентификаторы виртуальных сетей). Хост 1 и хост 2 находятся в одном офисе (VLAN 10), а хост 3 и хост 4 — в другом (VLAN 20).

*VLAN по портам (Access) - (Эмулятор: [miminet.ru](https://miminet.ru/web_network?guid=865d4618-0b97-45b5-a2af-392e15670f76))*

**Результат:** Хосты из разных VLAN не могут обмениваться данными даже на канальном уровне. Пакеты, отправленные на широковещательный MAC-адрес (`FF:FF:FF:FF:FF:FF`), не выходят за пределы своего VLAN.

---

## 2. Access VLAN (VLAN по портам)

**Access VLAN** — режим порта коммутатора, при котором порт принадлежит только одному VLAN. Все кадры, проходящие через такой порт, считаются принадлежащими этому VLAN.

**Алгоритм работы:**
1. Коммутатор получает кадр на Access-порт.
2. Определяет VLAN ID порта (например, порт 1 → VLAN 10, порт 2 → VLAN 10).
3. Коммутирует кадр только в рамках этого VLAN ID.

Если запустить `ping` от хоста 1 к хосту 2, то даже широковещательные пакеты (чей MAC-адрес назначения равен `FF:FF:FF:FF:FF:FF`) не попадут на хосты 3 и 4. Путём назначения VLAN ID на портах коммутатора мы создаём виртуальную сеть. И пакеты из одной виртуальной сети не будут попадать в другую.

> **Важно:** Можно представить, что внутри физического коммутатора создаются **виртуальные коммутаторы** для каждого VLAN. Трафик коммутируется только внутри этих виртуальных устройств. VLAN по портам также называют **Port-Based VLAN**.

---

## 3. Trunk VLAN (VLAN по тегам/меткам)

Если хосты из одного VLAN подключены к **разным** коммутаторам, возникает необходимость передавать трафик нескольких VLAN по одному физическому соединению.

*VLAN по портам (Access) на двух коммутаторах - (Эмулятор: [miminet.ru](https://miminet.ru/web_network?guid=e43b978b-6292-4619-8bec-0c82a75e004d))*

В этом примере для решения задачи потребовалось бы сделать два соединения между коммутаторами `l2sw1` и `l2sw2`: по одному соединению для VLAN 10, по-другому для VLAN 20. Но это избыточно, так как порт на коммутаторе — это не бесплатное удовольствие.

**Решение:** Использовать **Tagging VLAN** (тегированный VLAN) по стандарту **IEEE 802.1Q**. В Ethernet-кадр вставляется дополнительный заголовок (тег) размером **4 байта**, который содержит информацию о VLAN ID.

### Структура заголовка 802.1Q

Заголовок 802.1Q имеет размер 4 байта и помещается между полями MAC-адрес источника и поле Тип/Длина.

| **Поле** | **Размер** | **Назначение** |
| :--- | :--- | :--- |
| `TPID` | 16 бит | Tag Protocol Identifier. Всегда `0x8100`, указывает на использование 802.1Q. Это поле размещается прямо в том месте, где обычно размещается поле Тип/Длина. Таким образом, принимающая сторона может сразу определить, что имеет дело с заголовком 802.1Q. |
| `TCI` | 16 бит | Tag Control Information. Включает: |
| `PCP` | 3 бита | Priority Code Point (IEEE 802.1p) — приоритет трафика (0–7). Число 0 соответствует наименьшему приоритету, а 7 — наивысшему. |
| `DEI` | 1 бит | Drop Eligible Indicator — может быть отброшен при перегрузке. |
| `VID` | 12 бит | VLAN Identifier — номер VLAN (до 4096). Именно это поле позволяет прямо в пакете передать информацию о его принадлежности к конкретному VLAN ID. |
*Таб. 1. Структура заголовка 802.1Q.*

*VLAN по тегам (Trunk) — схема сети - (Эмулятор: [miminet.ru](https://miminet.ru/web_network?guid=1ccd87d4-a74f-485e-a95e-e1111c041fc7))*

**Trunk-порт** — порт коммутатора, который может передавать трафик нескольких VLAN. При отправке кадра на Trunk-порт коммутатор **добавляет VLAN-тег**, а при получении — **удаляет** его. Процесс удаления тега называется **терминацией**.

**Алгоритм работы Trunk (пошагово):**
1. Хост 1 отправляет пакет на хост 2.
2. Коммутатор 1 получает пакет от хоста 1, смотрит настройку порта (Access, VLAN ID = 10) и назначает данный пакет в VLAN 10.
3. Пакету на коммутаторе 1 разрешено коммутировать только между портами, которым разрешено работать в VLAN 10. На основе таблицы коммутации пакет нужно отправить в порт, который подключен к коммутатору 2.
4. Так как порт между коммутатором 1 и 2 сконфигурирован как Trunk, коммутатор 1 перед отправкой такого пакета добавляет в пакет тег (Tag) с информацией о VLAN (в нашем случае VLAN ID = 10).
5. Коммутатор 2 получает на Trunk-порт пакет с VLAN ID = 10, значит, данный пакет можно коммутировать только в порты, которые принадлежат VLAN ID = 10.
6. На основе таблицы коммутации пакет нужно отправить в порт, куда подключён хост 2.
7. Хост 2 подключён на Access-порт с VLAN ID = 10, значит, перед отправкой пакета тег удаляется (терминация).

## 4. Маршрутизация между VLAN

VLAN позволяют разделить один большой широковещательный домен на несколько небольших. Например, сеть компании на 1000 хостов можно разделить на отделы: бухгалтерия, серверная, разработчики и тестировщики. Без VLAN был бы один большой широковещательный домен, что вызывало бы проблемы с широковещательными пакетами и безопасностью.

С другой стороны, хосты из разных VLAN должны иногда взаимодействовать (например, разработчики и тестировщики хотят иметь доступ к серверам). Это приводит к необходимости маршрутизировать пакеты между различными VLAN. Для маршрутизации пакетов используют маршрутизатор.

### 4.1. Маршрутизация через физические интерфейсы

Маршрутизация между VLAN - *(Эмулятор: [miminet.ru](https://miminet.ru/web_network?guid=02521bae-d323-4384-af7c-47f8314607bb))*

**Схема:** К коммутатору подключено 2 хоста и 1 маршрутизатор:
- Хост 1 находится в VLAN 10 (Access-порт).
- Хост 2 находится в VLAN 20 (Access-порт).
- Маршрутизатор 1 подключён к двум портам коммутатора. Один порт находится в VLAN 10, другой в VLAN 20. Оба порта настроены как Access-порт.

Таким образом, маршрутизатор одновременно подключён сразу к двум VLAN:
- Интерфейс `192.168.1.1/24` подключён к порту коммутатора с VLAN 10.
- Интерфейс `10.0.0.1/24` подключён к порту коммутатора с VLAN 20.

**Алгоритм работы (пошагово):**
1. Хост 1 отправляет пакет хосту 2.
2. Пакет придёт на порт коммутатора с VLAN 10, значит, коммутация может выполняться только между портами с VLAN 10.
3. Коммутатор отправит пакет на маршрутизатор 1 через порт, который находится в VLAN 10.
4. Маршрутизатор получит пакет на интерфейс с IP-адресом `192.168.1.1`: разберёт IP-заголовок, определит, что IP-адрес получателя (`10.0.0.2`) не его, и будет этот пакет маршрутизировать дальше. На основе таблицы маршрутизации определит, что пакет можно отправить во второй интерфейс с IP-адресом `10.0.0.1`.
5. Маршрутизатор отправит пакет в сеть.
6. Пакет придёт на порт коммутатора с VLAN 20, т.е. коммутация может выполняться только между портами с VLAN 20.
7. Коммутатор отправит пакет хосту 2.

**Недостаток:** Занимает два порта на коммутаторе и два порта на маршрутизаторе.

### 4.2. Маршрутизация через сабинтерфейсы (ROAS — Router-on-a-stick)

Если маршрутизатор поддерживает создание сабинтерфейсов, можно обойтись одним физическим соединением.

**Сабинтерфейс** — это виртуальный интерфейс сетевого уровня (L3), который умеет отправлять и получать пакеты с VLAN-тегом (IEEE 802.1Q).

Маршрутизация между VLAN через сабинтерфейсы - *(Эмулятор: [miminet.ru](https://miminet.ru/web_network?guid=0f13773b-0ec4-45d4-85f7-ba91330fab08))*

**Настройка:**
- У маршрутизатора теперь одно физическое соединение вместо двух.
- Настроены два сабинтерфейса:
  - Сабинтерфейс `192.168.1.1/24` работает с VLAN 10.
  - Сабинтерфейс `10.0.0.1/24` работает с VLAN 20.
- На коммутаторе порт, подключённый к маршрутизатору, сконфигурирован как Trunk.

**Алгоритм работы (пошагово):**
1. Хост 1 отправляет пакет хосту 2.
2. Пакет придёт на порт коммутатора с VLAN 10, значит, коммутация может выполняться только между портами с VLAN 10.
3. Коммутатор отправит пакет на маршрутизатор 1 через Trunk-порт, который находится в VLAN 10 и 20. Коммутатор добавит VLAN-тег с указанием VLAN ID = 10.
4. Маршрутизатор получит пакет с VLAN-тегом, у которого VLAN ID = 10, значит, он будет обрабатываться сабинтерфейсом с IP-адресом `192.168.1.1`. Затем маршрутизатор:
   - разберёт IP-заголовок;
   - определит, что IP-адрес получателя (`10.0.0.2`) не его и будет этот пакет маршрутизировать дальше;
   - на основе таблицы маршрутизации определит, что пакет можно отправить в сабинтерфейс с IP-адресом `10.0.0.1`;
   - отправит пакет в сеть, при этом добавит в пакет VLAN-тег со значением VLAN ID = 20.
5. Пакет с VLAN ID = 20 придёт на Trunk-порт коммутатора, т.е. коммутация может выполняться только между портами с VLAN 20.
6. Коммутатор отправит пакет хосту 2.

**Преимущество:** Одно физическое соединение вместо двух. Такую схему подключения называют **ROAS (Router-on-a-stick)**.

> **Важно:** Используя VLAN и маршрутизацию между VLAN, можно не только уменьшить широковещательный домен, но и сделать сеть более безопасной (например, добавив правила фильтрации трафика на маршрутизаторе).

## 5. Native VLAN

Когда вы включаете VLAN на коммутаторе, у него появляется специальный **Native VLAN**. Это VLAN ID по умолчанию (обычно VLAN ID = 1).

**Зачем он нужен:**
1. Коммутаторы не только переправляют чужие пакеты, но и генерируют свои (например, STP — Spanning Tree Protocol, протокол для обнаружения и устранения петель в топологии Ethernet). Во время работы STP коммутаторы обмениваются специальными STP-пакетами. Если настроен Trunk-порт с VLAN ID = 10 и 20, коммутатору нужно отправить STP-пакет, но какой VLAN ID ставить в тег, 10 или 20?
2. В этой ситуации хочется просто отправить пакет без VLAN-тега. Если на Trunk-порт приходит пакет без тега, такой пакет автоматически приписывается к **Native VLAN**.

**Другой случай:** Если вы только что включили VLAN на коммутаторе, все порты автоматически будут в режиме Access и будут принадлежать Native VLAN. Это нужно, чтобы в момент включения VLAN на коммутаторе подключенные к этому коммутатору хосты продолжали работать.

**Вывод:** Коммутаторы, на которых включён VLAN, могут спокойно общаться друг с другом, используя Native VLAN, и не переживать о том, какой VLAN ID ставить в тег.

> **Рекомендация:** При настройке VLAN имеет смысл не использовать Native VLAN (VLAN 1) для пользовательского трафика. Лучше использовать VLAN ID, отличные от Native VLAN, чтобы отсечь нетегированные пакеты на Trunk-портах в создаваемых VLAN. Это повышает безопасность сети.

## 6. VLAN и MTU

Как известно из стандарта, максимальный размер Ethernet-кадра — **1518 байт**:
- 14 байт Ethernet-заголовок.
- До 1500 байт данных.
- 4 байта CRC32 (контрольная сумма, FCS).

Если мы включим VLAN-тег, он добавит ещё 4 байта. В итоге максимальный размер Ethernet-кадра вырастет до **1522 байт**.

**Вопрос:** Нужно ли при включении VLAN-тега изменять значение MTU (Maximum Transmission Unit)?

**Ответ:** Нет. Уже очень давно коммутаторы и сетевые карты по умолчанию могут принимать пакеты большей длины, чем 1518 или 1522 байта (поддержка Jumbo Frames).

> **Важно:** Разница между 1514, 1518 и 1522 байтами:
> **1514 байт** — размер Ethernet-кадра, который видит операционная система (без учёта FCS).
> **1518 байт** — полный размер Ethernet-кадра на физическом уровне (с учётом FCS).
> **1522 байта** — размер кадра с VLAN-тегом (1518 + 4 байта тега).

## 7. Разбор сложных инженерных ситуаций и аномалий

### 7.1. Могут ли хосты из разных VLAN общаться напрямую?

**Вопрос:** *Можно ли отправлять пакеты хосту из другой VLAN, зная его MAC-адрес?*

**Технический ответ:** **Нет.** Отправить кадр можно, но он не пройдёт через коммутатор, так как он коммутируется только в рамках одного VLAN. Для общения между VLAN необходима маршрутизация (L3). На канальном уровне VLAN полностью изолируют трафик.

### 7.2. Можно ли подделать VLAN-тег для доступа к другой VLAN?

**Вопрос:** *Если я вставлю VLAN-тег в свой кадр, смогу ли я попасть в другую VLAN?*

**Технический ответ:** Если порт коммутатора настроен как **Access**, он отбросит любой кадр с тегом. Если порт настроен как **Trunk**, он принимает тегированные кадры, но только те, которые разрешены в конфигурации (имеются в списке разрешённых VLAN). Таким образом, подделать тег для доступа к чужой VLAN практически невозможно.

### 7.3. Почему у хостов из разных VLAN могут быть одинаковые IP-адреса?

VLAN изолируют канальный уровень, поэтому IP-адреса могут повторяться в разных VLAN без конфликтов. Это допустимо, так как маршрутизация между ними происходит через L3, где используются уникальные сети. Например, в VLAN 10 может быть хост `10.0.0.1`, и в VLAN 20 тоже может быть хост `10.0.0.1` — это не вызовет конфликта, так как они находятся в разных широковещательных доменах.

### 7.4. Ограничение по количеству VLAN

Поле `VID` в заголовке 802.1Q имеет размер **12 бит**, поэтому максимальное количество VLAN — **4096** (от 0 до 4095). VLAN ID 0 и 4095 зарезервированы.

### 7.5. Разница между 1514, 1518 и 1522 байтами

- **1514 байт:** Размер Ethernet-кадра, который видит операционная система (без учёта FCS и преамбулы).
- **1518 байт:** Полный размер Ethernet-кадра на физическом уровне (с учётом FCS).
- **1522 байта:** Размер кадра с VLAN-тегом (1518 + 4 байта тега).

## 8. Расширенный ИБ-контекст и инженерный FAQ

### 1. Векторы атак и уязвимости

- **VLAN Hopping (Double Tagging):** Злоумышленник может использовать двойное тегирование для проникновения в другую VLAN. Первый тег удаляется коммутатором, второй остаётся и позволяет пакету попасть в целевую VLAN. Защита: отключение DTP и использование VLAN 1 только для Native VLAN.
- **Switch Spoofing:** Атака, при которой злоумышленник эмулирует Trunk-порт и получает доступ к трафику всех VLAN. Защита: отключение DTP и статическая настройка Trunk-портов.
- **VLAN Access Control:** Неправильная настройка VLAN может привести к утечке данных между арендаторами в дата-центре.
- **STP Manipulation:** Атаки на протокол STP могут вызывать широковещательные штормы и дестабилизацию сети.
- **MAC Flooding (CAM Table Overflow):** Переполнение таблицы коммутации может привести к тому, что коммутатор начнёт отправлять весь трафик на все порты, что позволит злоумышленнику перехватывать трафик из разных VLAN (если они не изолированы на L3).

### 2. Инженерный FAQ (На основе разбора дискуссий)

- **Вопрос:** *Куда вставляется VLAN-тег в Ethernet-кадре?*
- **Технический ответ:** **Между полями Source MAC и Type/Length.**

- **Вопрос:** *Как обеспечивается общение между хостами из разных VLAN?*
- **Технический ответ:** **С помощью маршрутизатора с сабинтерфейсами и подключением к Trunk-порту коммутатора (ROAS — Router-on-a-stick).**

- **Вопрос:** *Какой VLAN используют коммутаторы для общения между собой?*
- **Технический ответ:** **Native VLAN** (по умолчанию — VLAN 1).

- **Вопрос:** *На сколько байт увеличивается Ethernet-кадр при использовании Access VLAN?*
- **Технический ответ:** **На 0 байт.** Access VLAN не добавляет тег, он просто назначает порту VLAN ID.

- **Вопрос:** *На сколько байт увеличивается Ethernet-кадр при использовании Trunk VLAN (802.1Q)?*
- **Технический ответ:** **На 4 байта** (добавляется тег 802.1Q).

- **Вопрос:** *Для чего служит идентификатор кадра (tag)?*
- **Технический ответ:** **Содержит информацию о VLAN ID.**

- **Вопрос:** *Можно ли использовать одинаковые IP-адреса в разных VLAN?*
- **Технический ответ:** **Да.** VLAN изолирует канальный уровень, поэтому IP-адреса могут повторяться без конфликтов.

- **Вопрос:** *Сколько VLAN можно создать?*
- **Технический ответ:** **До 4096** (так как поле VID имеет размер 12 бит).

- **Вопрос:** *Что такое ROAS?*
- **Технический ответ:** **Router-on-a-stick** — схема, при которой маршрутизатор подключается к коммутатору одним физическим Trunk-портом, а маршрутизация между VLAN осуществляется через сабинтерфейсы.

---

## Заключение

VLAN — это фундаментальная технология сегментации сетей на канальном уровне. Она позволяет изолировать трафик, повышать безопасность, уменьшать широковещательные домены и упрощать администрирование. Понимание режимов Access/Trunk, стандарта 802.1Q, маршрутизации между VLAN (включая ROAS) и Native VLAN является обязательным для любого сетевого инженера. Правильная настройка VLAN также критична для обеспечения информационной безопасности, так как неправильная конфигурация может привести к утечкам данных и атакам типа VLAN Hopping.
