# Тема 5.4: ARP Proxy (Proxy ARP)

Изучение техники использования ARP-протокола, позволяющей объединять два физически разделённых сегмента сети в единую IP-подсеть, обеспечивая прозрачное взаимодействие хостов без настройки маршрутизации на конечных устройствах.

---

## 1. ARP Proxy — Базовые свойства

- **Уровень OSI:** Канальный уровень (L2) — работа с ARP-протоколом.
- **Сущность данных (PDU):** Кадр (Frame) / ARP-сообщение.
- **Принцип работы:** ARP Proxy (Proxy ARP — прокси-ARP) — это техника, при которой маршрутизатор отвечает на ARP-запросы от имени хостов из другой подсети, подставляя свой MAC-адрес. Это позволяет объединить два физически разделённых сегмента сети в одну логическую IP-подсеть.

**Определение:** ARP Proxy — техника использования ARP-протокола, позволяющая объединить две не связанные на канальном уровне сети в одну. Хосты, находящиеся в этих сетях, могут использовать адреса из одной IP-подсети и обмениваться трафиком между собой без использования маршрутизатора (как им кажется).

---

## 2. Принцип работы ARP Proxy

Рассмотрим схему, на которой изображены два хоста — **Хост A** и **Хост B**, которые находятся на канальном уровне в разных сегментах сети. Между ними стоит маршрутизатор. На хостах не настроен шлюз по умолчанию. Маски подсетей на маршрутизаторе (`/24`) и на хостах (`/8`) отличаются.

**Настройки сети:**
- Хост A: IP-адрес `10.0.1.2/8`.
- Хост B: IP-адрес `10.0.2.2/8`.
- Маршрутизатор: Интерфейс 1 — `10.0.1.1/24`, Интерфейс 2 — `10.0.2.1/24`. На обоих интерфейсах включён **Proxy ARP**.

### Алгоритм работы ARP Proxy (пошагово)

| **Шаг** | **Что происходит** |
| ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1       | Хост A хочет отправить данные хосту B. Так как на хосте A IP-адрес `10.0.1.2` с маской `/8`, он считает, что хост B с IP-адресом `10.0.2.2` находится с ним в одной сети (`10.0.0.0/8`). |
| 2       | Хосту A необходимо узнать MAC-адрес хоста B. Он отправляет **ARP-запрос** в сеть, где ищет MAC-адрес для IP `10.0.2.2`. |
| 3       | Маршрутизатор получает ARP-запрос, но не перенаправляет его, так как получатель находится в другой сети. Однако, если на маршрутизаторе включён **ARP Proxy**, он отправляет хосту A **ARP-ответ**, в котором подставляет **свой MAC-адрес** (интерфейса `10.0.1.1`). |
| 4       | Хост A получает ARP-ответ и создаёт в своём ARP-кэше запись: IP-адрес `10.0.2.2` соответствует MAC-адресу маршрутизатора (`10.0.1.1`). |
| 5       | Теперь хост A может отправлять данные на MAC-адрес маршрутизатора. Пакеты направляются на маршрутизатор, как будто он и есть хост B. |
| 6       | Маршрутизатор получает пакет от хоста A, смотрит на IP-адрес получателя (`10.0.2.2`) и понимает, что пакет нужно перенаправить на хост B. |
| 7       | Маршрутизатор заглядывает в свой ARP-кэш, не находит запись для `10.0.2.2` и отправляет **ARP-запрос** в сеть B для поиска MAC-адреса хоста B. |
| 8       | Хост B с IP-адресом `10.0.2.2` получает ARP-запрос, сверяет искомый IP со своим. Так как IP-адреса равны, хост B отправляет **ARP-ответ** со своим MAC-адресом. |
| 9       | Маршрутизатор, получив ARP-ответ, добавляет запись в свой ARP-кэш и теперь может пересылать данные от хоста A для хоста B. |
| 10      | Хост B аналогичным образом считает, что хост A находится с ним в одной сети. Когда хосту B необходимо узнать MAC-адрес хоста A, он отправляет ARP-запрос в сеть. Маршрутизатор получает ARP-запрос и, так как включён ARP Proxy, отвечает хосту B своим MAC-адресом. |
| 11      | Маршрутизатор перенаправляет ответные пакеты от хоста B к хосту A по тому же принципу. |

*Таб. 1. Алгоритм работы ARP Proxy.*

**Итог:** Таким образом, Proxy ARP помогает **прозрачно** объединить два физических сегмента в один. Хосты A и B «думают», что находятся в одной сети, и обмениваются данными, даже не подозревая о существовании маршрутизатора.

## 3. Разбор сложных инженерных ситуаций и аномалий

### 3.1. Почему ARP Proxy не перенаправляет ARP-запросы в другую сеть?

**Вопрос:** *Почему маршрутизатор не перенаправляет ARP-запрос от хоста A в сеть B?*

**Технический ответ:** ARP-запросы являются широковещательными кадрами канального уровня (L2). Маршрутизаторы по умолчанию **не перенаправляют широковещательный трафик** между сетями. ARP Proxy позволяет маршрутизатору **ответить на ARP-запрос самостоятельно**, подставив свой MAC-адрес, вместо того чтобы перенаправлять запрос.

### 3.2. В чём опасность ARP Proxy?

**Вопрос:** *Какие риски связаны с использованием ARP Proxy?*

**Технический ответ:** Основная опасность — **подмена ARP-ответов (ARP Spoofing)**. Злоумышленник в локальной сети может подменить ARP-ответы и выдать свой MAC-адрес как MAC-адрес маршрутизатора. В случае с ARP Proxy это может привести к **MITM-атаке**, когда весь трафик между двумя сегментами будет проходить через злоумышленника.

### 3.3. Когда ARP Proxy используется на практике?

**Вопрос:** *В каких реальных сценариях применяется ARP Proxy?*

**Технический ответ:** Основные сценарии:
- **Разделение сети через VLAN:** При использовании VLAN с ARP Proxy можно объединить хосты из разных VLAN в одну IP-подсеть без настройки маршрутизации на хостах.
- **Миграция сетей:** При переходе с одной топологии на другую ARP Proxy позволяет временно сохранить связность без перенастройки всех хостов.
- **Объединение старых и новых сегментов:** Когда часть устройств физически находится в разных сегментах, но должна логически быть в одной подсети.

---

## 4. Границы доменов и лимиты масштабирования

- **Broadcast Domain (Широковещательный домен):** ARP Proxy работает на границе двух широковещательных доменов. Хосты A и B находятся в разных широковещательных доменах, но логически объединены в одну IP-подсеть через ARP Proxy.
- **Лимиты масштабирования:** ARP Proxy не масштабируется на большие сети. При большом количестве хостов ARP-кеш маршрутизатора может переполняться, что приведёт к снижению производительности.
- **Зависимость от ARP-кеша:** Работа ARP Proxy критически зависит от корректности ARP-кеша маршрутизатора. Если записи в ARP-кеше устареют или будут подменены, связность нарушится.

## 5. Расширенный ИБ-контекст и инженерный FAQ

### 1. Векторы атак и уязвимости

- **ARP Spoofing (MITM):** Злоумышленник в одном из сегментов может подменить ARP-ответы и выдать свой MAC-адрес как MAC-адрес маршрутизатора. Весь трафик между сегментами будет проходить через атакующего.
- **ARP Cache Poisoning:** Злоумышленник может отравить ARP-кеш маршрутизатора, подменив записи для хостов A и B, что приведёт к перенаправлению трафика на атакующего или к DoS-атаке.
- **Обход политик безопасности:** ARP Proxy может использоваться для обхода VLAN-изоляции, если маршрутизатор неправильно сконфигурирован.

### 2. Инженерный FAQ (На основе разбора дискуссий)

- **Вопрос:** *Для чего используется ARP Proxy?*
- **Технический ответ:** ARP Proxy используется для объединения двух физически разделённых сегментов сети в одну логическую IP-подсеть без необходимости настройки шлюза по умолчанию на конечных хостах.

- **Вопрос:** *Чем отличается ARP Proxy от обычного ARP?*
- **Технический ответ:** Обычный ARP работает в пределах одного широковещательного домена. ARP Proxy позволяет маршрутизатору отвечать на ARP-запросы от имени хостов из других сегментов, подставляя свой MAC-адрес.

- **Вопрос:** *Какую маску подсети должны иметь хосты для работы ARP Proxy?*
- **Технический ответ:** Хосты должны иметь маску, объединяющую их в одну IP-подсеть (в примере — `/8`). Маршрутизатор использует другую маску (`/24`) для каждого интерфейса.

- **Вопрос:** *Можно ли использовать ARP Proxy в домашней сети?*
- **Технический ответ:** Обычно нет. ARP Proxy используется в корпоративных сетях при объединении сегментов, особенно при работе с VLAN или при миграции сетей.

- **Вопрос:** *Включается ли ARP Proxy по умолчанию на маршрутизаторах?*
- **Технический ответ:** Нет. ARP Proxy требует явного включения на интерфейсах маршрутизатора, так как может создавать риски безопасности.

---

## Заключение

ARP Proxy — это полезная, но редко используемая техника, позволяющая объединять физически разделённые сегменты сети в одну логическую IP-подсеть. Она упрощает миграцию сетей и работу с VLAN, но требует осторожности из-за рисков ARP-спуфинга и неправильной конфигурации. Понимание принципов ARP Proxy важно для диагностики сетевых проблем и проектирования гибких сетевых архитектур.
