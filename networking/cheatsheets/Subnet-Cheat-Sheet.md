## ФОРМУЛЫ ДЛЯ РАСЧЁТОВ

| Что считать | Формула |
|---|---|
| Количество адресов | 2^(32 - префикс) |
| Полезных хостов | 2^(32 - префикс) - 2 |
| Количество подсетей при разбиении /X -> /Y | 2^(Y - X) |
| Маска в десятичном виде | См. таблицу выше |

## КАК БЫСТРО СЧИТАТЬ В УМЕ

### Способ 1: По формуле
Количество адресов = 2^(32 - префикс)

Пример: /27 -> 32 - 27 = 5 -> 2^5 = 32 адреса

### Способ 2: По шагам (от /24)

| Префикс | Адресов | Во сколько раз больше /24 |
|---|---|---|
| /24 | 256 | 1x (база) |
| /23 | 512 | 2x |
| /22 | 1024 | 4x |
| /21 | 2048 | 8x |
| /20 | 4096 | 16x |
| /19 | 8192 | 32x |
| /18 | 16384 | 64x |
| /17 | 32768 | 128x |
| /16 | 65536 | 256x |

Правило: Уменьшаешь префикс на 1 -> адресов становится в 2 раза больше.

---

## КАК НАЙТИ ГРАНИЦЫ СЕТИ (пошаговый пример)

Пример: 192.168.1.50/26

1. Сколько адресов? /26 -> 32 - 26 = 6 -> 2^6 = 64 адреса
2. Шаг сетей: 256 / 64 = 4 (сети идут с шагом 4 в последнем октете)
3. Границы: 192.168.1.0, 192.168.1.64, 192.168.1.128, 192.168.1.192
4. Твой адрес 50 попадает в диапазон 0-63 -> сеть 192.168.1.0/26

## КАК ОПРЕДЕЛИТЬ МАСКУ ПО IP-АДРЕСУ (без таблицы)

### Способ 1: По количеству хостов в сети

Если ты знаешь, сколько хостов должно быть в сети, выбираешь маску по таблице:

| **Нужно хостов** | **Берёшь маску** | **Всего адресов** |
|---|---|---|
| 1 | /32 | 1 |
| 2 | /30 (или /31) | 4 (или 2) |
| 3–6 | /29 | 8 |
| 7–14 | /28 | 16 |
| 15–30 | /27 | 32 |
| 31–62 | /26 | 64 |
| 63–126 | /25 | 128 |
| 127–254 | /24 | 256 |
| 255–510 | /23 | 512 |

---

### Способ 2: По маске в десятичном виде (если тебе дали маску)

Если тебе дали маску в виде `255.255.255.0`, ты можешь перевести её в префикс:

1. Смотришь на каждый байт маски
2. Считаешь, сколько единиц в каждом байте

| **Значение байта** | **Сколько единиц** |
|---|---|
| 0 | 0 |
| 128 | 1 |
| 192 | 2 |
| 224 | 3 |
| 240 | 4 |
| 248 | 5 |
| 252 | 6 |
| 254 | 7 |
| 255 | 8 |

**Пример:** `255.255.255.224`
- Первый байт: 255 → 8
- Второй байт: 255 → 8
- Третий байт: 255 → 8
- Четвёртый байт: 224 → 3
- Всего: 8 + 8 + 8 + 3 = **27** → маска `/27`

---

### Способ 3: По CIDR-записи (если тебе дали префикс)

Если тебе дали запись типа `192.168.1.0/24`, то маска уже указана — это `/24`.

**Что делать:**
1. Смотришь на число после слеша
2. Это и есть маска

---

### Способ 4: Если ты знаешь адрес сети и широковещательный адрес

Если ты знаешь, что:
- Адрес сети: `192.168.1.0`
- Широковещательный адрес: `192.168.1.255`

То:
1. Количество адресов: `255 - 0 + 1 = 256`
2. Биты под хосты: `log2(256) = 8`
3. Префикс: `32 - 8 = 24` → маска `/24`

---

### Способ 5: Если ты знаешь, что адрес принадлежит частной сети

| **Частная сеть** | **Маска** |
|---|---|
| `10.x.x.x` | `/8` |
| `172.16.x.x` – `172.31.x.x` | `/12` |
| `192.168.x.x` | `/16` |

Но это только если сеть **не разбита** на подсети. В реальности админ мог разбить `192.168.1.0/24` на `192.168.1.0/25` и `192.168.1.128/25`.

## БЫСТРЫЙ АЛГОРИТМ (по шагам)

| **Шаг** | **Действие** |
|---|---|
| 1 | Посмотри на IP-адрес |
| 2 | Если есть запись типа `/24` — это маска |
| 3 | Если дана маска типа `255.255.255.0` — переведи в префикс |
| 4 | Если знаешь адрес сети и широковещательный — посчитай |
| 5 | Если ничего нет — спроси у администратора сети |
